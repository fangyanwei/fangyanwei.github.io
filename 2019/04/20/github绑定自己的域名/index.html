github绑定自己的域名

发表于 2019-04-20 | 分类于 web
字数统计: 321 | 阅读时长 ≈ 1

第一种方式：

1.在自己的仓库设置里面添加自己购买的域名

2.在阿里云服务器域名的DNS解析记录里添加两条记录。

此时会发现输入自己的域名可以跳转到自己的github pages了，但是这种方式有一个问题，如果你是用 hexo 框架搭建博客并部署到 Github Pages 上，每次hexo d -g后域名解析又不行了。此时我们就要用到第二种方式。

第二种方式

1.在source文件夹下新建一个文件CNAME（没有后缀），用文本编辑器打开首行加入xxx.com（你的域名，不用http://,也不用www),然后

hexo d -g

部署到github

2.同样在阿里云DNS解析记录里添加两条记录

先添加一个CNAME，主机记录写@，记录值填上你的xxx.github.io
再添加一个CNAME，主机记录写www,记录值xxx.github.io

3.等十分钟左右，刷新浏览器，用你自己域名访问下试试

第一种方式是用A记录，后面的记录值是github pages的ip地址，但是有时候ip地址会更改，导致解析不正确，所以还是推荐用CNAME别名记录方式好些。
