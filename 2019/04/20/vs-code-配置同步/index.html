vs code 配置同步

发表于 2019-04-20 | 分类于 web
字数统计: 532 | 阅读时长 ≈ 2

VS Code默认是不支持同步的，如果在两台以上的电脑特种不方便，安装的插件和配置又要重新来一遍。好在vs的插件特别多，使用Settings Sync借助github就可以解决这个问题。

首先你要有一个github帐号，没有的先申请一下

1.安装Settings Sync插件并重启VSCODE

2.2.重启后按快捷键 alt+shift+u (这里假设你第一次用)

它会弹出一个窗口对应的是github上面的创建个人gist的页面,如果未登录请先登录github.
然后Settings –> Developer settings —>Personal access Tokens

在Tokens Description描述里填上你token的用途，然后勾选gist.然后点击生成会有一个key。

3.切回到vscode,他会有个输入区,就是存放刚才生成的key

然后理论上他就开始对你本机的配置进行一个扫描上传了.至此上传工作完成.

接下来我们到另一台电脑上了下载配置.同样的先安装Settings Sync插件,并重新加载.

然后按快捷键alt+shift+d,就应该会弹出一个输入框,请在这里输入之前保存下来的key(GIST ID),回车后将会自动下载之前上传的配置.

那么下载完成后,你这台电脑修改了相关配置再次上传就好了.是不是感觉方便多了~

然而，我在使用的过程中，因为有个gist id 没记录起来导致在下载配置的时候提示错误。

补救方法

1.按F1，输入sync,这里有重置选项，点击重置它。

2.再按alt+shift+u，输入刚才生成的token

如果忘记了token,可以在C:\Users\<你的用户名>\AppData\Roaming\Code\User\syncLocalSettings.json 。搜索token即可找到token

3.输入gist id

这个id 要在上传配置的电脑vs code里操作，在VS Code界面，点击左下角小齿轮，选择设置后，自动打开“User Setting”,搜索sync.gist，右侧等号后的一串字符为gist id。

此时就可以下载同步成功了。
