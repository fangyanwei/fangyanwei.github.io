1.问题

如图，点击表情插入输入框，超过其宽度后没被裁剪。

2.问题解析

在点击表情的时候是这样设置文本的

chatInput.changeText(chatInput.text+emotionObj.pure);

看下官方对这个接口的定义：

接口已经很明确的说了，不进行排版计算。所以最简单的解决方案是用text接口，这样设置之后就会重新排版，不会超出。

chatInput.text = chatInput.text+emotionObj.pure;

可以看到设置text后，会在下一帧中调用typeset刷新

那为什么聊天的输入框就可以，好友这个输入框就不行呢？
印象中之前聊天输入也是有这个问题，后面有人提出说点击输入那个文本会跳来跳去的，当时没闹明白怎么回事就去改了。现在看来是因为我用的控件是TextInput，但是在代码我却把它定义成了Input，因为TextInput是继承自Label，它跟Text相比是Label有相对布局，所以强转成了Input后它就失去了自动布局的特性，表现上就会获得焦点跟失去焦点时文本的位置会抖。

public class TextInput extends Label

public class Input extends Text

public class Label extends UIComponent

3.总结

内容超过输入框的宽度没有裁剪的问题有两个方法解决：
1)用text方法替换changeText，这样能保证每次变化都重新计算一次布局，只是相比changeText牺牲点性能。
2)如果不想损耗性能，则要在输入框底下多一个背景，然后输入就不要背景了，调整基高度跟字号一样，这样在第
一次设置默认显示的文本时会设置其裁剪大小，保证超过后会被裁剪掉。
